Z-Index Stacking Context Solution

Problem Overview

The search dropdown was being covered by the floating chat window despite having a much higher z-index value (100000 vs 9999).

Search Dropdown Portal
z-index: 999999
Floating Chat Portal
z-index: 50000
TopMenuBar
z-index: 40
Normal Content
z-index: 1

Root Cause Analysis

  1. Stacking Context Trap: The search dropdown was trapped within its parent's stacking context (TopMenuBar with z-index: 40)
  2. Portal Competition: The floating chat used a portal with z-index: 9999, creating a separate stacking context
  3. Z-index Inheritance: Child elements cannot escape their parent's stacking context, regardless of z-index value

Solution Implementation FIXED

❌ Before (Broken)

// Search dropdown inside TopMenuBar <div className="z-[10000]"> <ProductSearchDropdown> <div className="z-[100000]"> // Still trapped in parent context </div> </ProductSearchDropdown> </div>

✅ After (Fixed)

// Search dropdown uses React Portal createPortal( <div style={{ zIndex: 999999, position: 'fixed' }}> // Escapes all parent contexts </div>, portalRoot )

Changes Made

  1. ✅ Modified ProductSearchDropdown.tsx to use React Portal
  2. ✅ Created dedicated portal root with z-index: 999999
  3. ✅ Updated dropdown positioning to use fixed positioning
  4. ✅ Adjusted FloatingChatContainer portal z-index to 50000
  5. ✅ Removed unnecessary z-index from all TopMenuBar components
  6. ✅ Created centralized z-index.css for consistent management

Z-Index Hierarchy (Final)

/* Established Z-Index Hierarchy */ --z-base: 1; /* Normal content */ --z-header: 40; /* Headers and navigation */ --z-floating-chat: 50000; /* Floating chat window */ --z-search-dropdown: 999999; /* Search dropdown (highest) */

Testing Instructions

  2. Click on the chat to make it float (if not already floating)
  3. Type in the search bar to trigger the dropdown
  4. The search dropdown should now appear above the floating chat window
  5. Test across all templates (dark-tech, modern-minimal, pot-palace, rasta-vibes)

Benefits of Portal Solution

  • ✅ Escapes Stacking Context: Portals render outside the parent DOM hierarchy
  • ✅ Predictable Z-Index: Direct control over stacking order
  • ✅ Performance: No need for excessive z-index values throughout the app
  • ✅ Maintainable: Centralized z-index management in CSS
  • ✅ Cross-Template: Works consistently across all templates
